## Point Value Guidelines

### Satisfying Scores

Point values affect how players feel:

| Low Points | High Points |
|------------|-------------|
| More strategic | More exciting |
| Careful play | Big numbers |
| Subtle progress | Visible progress |

### Common Approaches

| Style | Example | Effect |
|-------|---------|--------|
| Low base | 10 per action | Scores feel earned |
| Medium base | 100 per action | Balanced feel |
| High base | 1000 per action | Big, exciting numbers |

## Feature Toggles

Some games have features you can enable or disable:

| Toggle | What It Controls |
|--------|------------------|
| Tutorial | Show/hide tutorial overlay |
| Sound Effects | Enable/disable audio |
| Haptics | Enable/disable vibration |
| Power-ups | Show/hide special abilities |

## Adjusting Rewards

If your game shows currency rewards:

* **Higher rewards** = More exciting, aspirational
* **Lower rewards** = More realistic preview
* **Match your game** = Authentic representation

## Why Review May Be Required

Gameplay values affect the overall experience:

| Quick Changes | Reviewed Changes |
|---------------|------------------|
| Feature toggles | Core point systems |
| Minor adjustments | Win/lose conditions |
| Tutorial visibility | Reward structures |

Review ensures your ad accurately represents your game.

## Testing Your Changes

After adjusting values:

1. Play through the full experience
2. Check if scores feel satisfying
3. Verify rewards make sense
4. Ensure the experience is fun

## Best Practices

### For Playable Ads

* **Generous points** - Make players feel successful
* **Achievable goals** - Set beatable targets
* **Clear feedback** - Show progress obviously
* **Rewarding finish** - End on a high note

### Avoid

* Impossible scores to reach
* Confusing point systems
* Rewards that seem unrealistic
* Features that confuse users
