# Custom Development

Custom development is when you need something that goes beyond the dashboard's built-in options.

## The Rule

> **Dashboard parameter = Adjustable**
> 
> **Complex change = Quote required**

Parameters in your dashboard are adjustable. Custom development is for complex tasks requiring code changes.

## Dashboard vs Custom Examples

| Dashboard Parameter | Custom Development |
|---------------------|-------------------|
| Change button color | Add new button type |
| Edit existing text | Add new text fields |
| Replace images | Generate map from image |
| Adjust timer display | Add new timer logic |
| Style HP bar | Add health mechanics |
| Edit end card text | Add new screen types |
| Update animation frames | Requires quote |

## Why the Difference?

### Dashboard Changes (Free)

When you make changes in the dashboard, you're adjusting values our system already knows about:
* The elements exist
* The framework is built
* You're just customizing settings

### Custom Development (Quote)

Custom development means creating something entirely new:
* Design planning
* Code development
* Quality testing
* Integration work
* Deployment

## What's Possible?

The good news: **almost anything is possible!**

If you can imagine it, we can probably build it:
* New game mechanics
* Special effects
* Unique interactions
* Platform integrations
* Advanced features

## How to Know If You Need Custom Work

Ask yourself:

1. **Does this parameter appear in my dashboard?**
   * Yes = You can adjust it
   * No = It may need custom development

2. **Is this a visual/UI change or game logic?**
   * UI styling = Usually adjustable
   * Game logic = Usually requires development

3. **Am I trying to add something completely new?**
   * Yes = Likely custom development
   * No = Check the dashboard first

## How to Request a Quote

### Step 1: Describe What You Want

Tell us:
* What should happen?
* When should it appear?
* How should it look?

### Step 2: Explain Why You Need It

Help us understand:
* What goal does this achieve?
* Why can't dashboard options work?
* How important is this?

### Step 3: Share References

The more context, the better:
* Screenshots of similar features
* Links to ads with features you like
* Sketches or mockups

### Step 4: Timeline

Let us know:
* When do you need this?
* Is there flexibility?
* Any hard deadlines?

## What Happens Next

1. We receive your request (within 1 business day)
2. We assess the work (determine scope and complexity)
3. You receive a quote (with cost and timeline)
4. You decide (no obligation to proceed)

## Quote Details

Your quote will include:

| Item | Description |
|------|-------------|

| Cost | Total price for the work |
| Timeline | How long it will take |
| Deliverables | What you'll receive |
